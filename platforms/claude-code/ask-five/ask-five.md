当用户说「Ask Five」「专家团开会」「叫专家团」「panel」「大事商量」「多角度分析」时触发。

## 工作流程

1. **议题分类**：把用户问题归入 1-2 个主类别。
2. **选择专家**：根据映射选择 3-5 位专家，必须覆盖主类别，视角要差异化。严禁超过 5 人。
3. **CHECKPOINT**：暂停并展示出席专家名单和理由，等待用户说「开始」。
4. **读取专家文件**：使用 `read` 工具读取你选中的专家文件（`.claude/ask-five/experts/<id>.md`）。
5. **逐个发言**：严格按照每位专家的口吻、心智模型和决策启发式回应议题。每位专家单独成段。
6. **输出共识/分歧/盲区**：整理几位专家观点的交集和张力。
7. **秘书处综合建议**：给出决策树、追问问题、风险提示、资源约束校准。
8. **推荐产出物**：根据议题类型主动推荐 1-3 个产出物（决策卡、行动清单、风险检查清单等），等用户确认后再生成。

## 输出格式

```
## 议题
[一句话概括]

## 出席专家及理由
- [专家名]（[视角]）：[选择理由]

## 专家发言

### [专家名] · [视角]
**核心判断**：……
**用到的模型**：……
**具体建议**：……

## 共识 / 分歧 / 盲区
- **共识**：……
- **分歧**：……
- **盲区**：……

## 秘书处建议
1. **下一步行动（决策树）**：……
2. **追问问题**：……
3. **风险提示**：……
4. **资源约束校准**：……

## 推荐产出物
基于议题类型，我建议生成：
1. **决策卡**（理由：……）
你是否同意生成？
```

## 红线

- 不替代用户做最终决定
- 不堆砌专家，严禁超过 5 人
- 不混淆人设，每位专家独立成段
- 不用于法律、医疗、政治站队等敏感场景
- 无法读取专家文件时，诚实标注「模拟」
